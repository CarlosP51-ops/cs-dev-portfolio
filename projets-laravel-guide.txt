1. BLOG MULTI-AUTEURS
================================================================================

Base de données : MySQL
Niveau : Intermédiaire

FONCTIONNALITÉS
---------------
- Authentification complète (register, login, email verification, password reset)
- 3 rôles : admin (tout gérer), auteur (ses articles), lecteur (commenter)
- Articles avec TinyMCE, slug auto-généré, image de couverture
- Système de tags et catégories avec filtres
- Commentaires avec modération (approuver/rejeter)
- Newsletter : abonnement email + envoi groupé via Queue

CONCEPTS LARAVEL COUVERTS
--------------------------

- Queues et Jobs (envoi emails en arrière-plan)
- Events et Listeners (ex: ArticlePublié → notifier abonnés)
- Observers (générer slug automatiquement)
- Scopes Eloquent (articles publiés, populaires)

---------------

3. PLATEFORME E-COMMERCE COMPLÈTE
================================================================================

Base de données : MySQL + Redis
Niveau : Avancé

FONCTIONNALITÉS
---------------
- Catalogue avec catégories, filtres prix/note/stock
- Panier persistant (session + base de données pour connectés)
- Checkout avec intégration Stripe (paiement carte + webhooks)
- Gestion commandes : en attente, confirmée, expédiée, livrée
- Factures PDF générées automatiquement (Laravel DomPDF)
- Emails transactionnels (confirmation commande, expédition)
- Gestion stock avec alertes quand stock bas
- Panel admin : produits, commandes, clients, revenus
- Système d'avis et notes sur les produits
- Codes promo avec règles (%, montant fixe, usage limité)

6. API REST + SPA (LARAVEL 11 + REACT)
================================================================================

Base de données : PostgreSQL + Redis
Niveau : Expert

FONCTIONNALITÉS
---------------
- Laravel comme backend pur (pas de Blade)
- Authentification Sanctum avec tokens
- CRUD complet avec validation poussée
- Pagination, filtres, tri via query params
- Upload fichiers avec validation MIME
- Rate limiting par endpoint
- Versioning API (v1, v2)
- Documentation auto avec Swagger/L5-Swagger
- Tests Feature complets (chaque endpoint testé)
- React comme frontend avec React Query pour les appels API

CONCEPTS LARAVEL COUVERTS
--------------------------
- API Resources (transformer les modèles en JSON propre)
- Form Requests avec règles de validation custom
- Rate Limiting avec RateLimiter facade
- Sanctum (tokens, abilities/scopes)
- Tests Feature avec RefreshDatabase
- Repository Pattern (séparer la logique métier)
- Service Layer (controllers fins, logique dans services)

STRUCTURE RECOMMANDÉE DU CODE
------------------------------

    Controllers/Api/V1/
    Resources/
    Requests/
  Services/
  Repositories/
  Interfaces/

POURQUOI MYSQL POUR LA PLUPART ?
---------------------------------
- Tu le connais déjà (ton portfolio tourne dessus)
- Parfait pour les données relationnelles classiques
- Très bien supporté par Laravel
- Hébergement gratuit facile (Railway, PlanetScale)

POURQUOI POSTGRESQL POUR L'API REST ?
--------------------------------------
- Plus robuste pour les applications professionnelles
- Meilleur support des types JSON avancés
- Compétence très demandée en entreprise
- Prépare aux environnements enterprise

POURQUOI REDIS ?
-----------------
- Cache du feed et des compteurs (likes, vues) → performances x10
- Queues pour les jobs en arrière-plan (emails, PDF, notifications)
- Sessions rapides
- Indispensable dès que l'app grossit

POURQUOI S3 POUR LES FICHIERS ?
---------------------------------
- Ne jamais stocker des vidéos ou images en local sur le serveur
- Le serveur peut redémarrer et perdre les fichiers
- S3 est scalable, fiable, et peu coûteux
- Cloudinary est une alternative gratuite pour les images

RÈGLES GÉNÉRALES SUR LES TYPES DE COLONNES
--------------------------------------------
- Foreign keys          → unsignedBigInteger
- Prix                  → decimal(10,2) — jamais float (imprécis)
- Valeurs fixes         → enum (status, role, type)
- Données structurées   → json (options, metadata, stack)
- Suppression douce     → soft deletes (deleted_at) sur données importantes
- Index                 → sur toutes les FK et colonnes fréquemment filtrées
- Pivot unique          → index unique sur les colonnes de pivot (follow, enrollment)
